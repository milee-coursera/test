AI in the Flow of Work for Product Managers
Reference: AI Spark 2026 presentation
AI in the Flow of Work teaches you to use AI the way your CTO does — not as a writing assistant, but as a collaborator that runs your data queries, drafts your stakeholder updates, and builds working prototypes. 5 skills. 19 tasks. All grounded in your actual Coursera workflows.

By the end you'll have: a prompt system that produces first-draft-ready outputs, a persistent AI context file so every session starts knowing your product area and stakeholders, live Databricks queries without copy-pasting, a weekly update pipeline that produces three audience-ready versions in under 15 minutes, and the ability to run pre-launch and production evals on your AI features. Every task is something you can do in the real world.

Capstone projects

Three optional capstone projects — each produces a real artifact you use in your actual job this week. Complete any one, or all three. Capstones run in your real Claude Code environment, not the simulator. An AI provides live feedback via screen share overlay.
Design principle: Every capstone artifact must be usable this week. If it can't be sent to a real stakeholder or run in a real meeting, it's not done.
Capstone A
Automated weekly update pipeline
Build a reusable pipeline that takes your raw weekly notes + Databricks data, and produces a stakeholder-ready update in three formats — exec summary, full team update, async Slack digest — in one session. Runs in under 10 minutes. You never format a weekly update from scratch again.
Deliverable 1Prompt library: templates for 3 audience formats (exec, team, async)
Deliverable 2Databricks MCP query that auto-pulls your weekly metrics into the pipeline
Deliverable 3Skill file: weekly-update.md — callable in one command next week
Deliverable 4Live test: run the pipeline on this week's actual notes. Output must be sendable.
Prerequisites: S1 (T1–T4) + S2 (T5–T7) + S3 (T8–T11). Est. time: 45–60 min.
Capstone B
Data-informed communication for multiple audiences
Take a real product insight from Databricks — a retention drop, funnel anomaly, or cohort comparison — and produce three communications from one query: exec brief, structured team analysis, and QBR-ready slide summary. Each version is audience-calibrated and stakeholder-ready.
Deliverable 1Databricks query surfacing a real insight from your product area
Deliverable 2Exec brief: 1 paragraph, implication-first
Deliverable 3Team analysis: data + interpretation + 3 recommended actions
Deliverable 4QBR-ready summary: visual-first framing as interactive HTML diagram
Prerequisites: S1 (T1–T4) + S3 (T8–T11) + S5 (T17–T19). Est. time: 45–60 min.
Capstone C
Vibe-coded working prototype
Build a working interactive prototype for a feature you're currently shaping — no design handoff, no engineering bandwidth. The prototype runs in the browser, is shareable as a link, and demonstrates the core interaction. You use it in your next stakeholder or solution-shaping session.
Deliverable 1One-paragraph spec: what the prototype demonstrates and who it's for
Deliverable 2Working prototype: clickable, browser-based, core interaction functional
Deliverable 3Light eval: 3 things tested, what passed, what you'd change before showing eng
Deliverable 4Shareable link (GitHub Pages) — ready to use in your next session
Prerequisites: S1 (T1–T4) + S4 (T12–T14). Est. time: 60–90 min.
